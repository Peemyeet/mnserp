# ความหมายของตาราง `market` ใน db_mns

## สรุปสำหรับทีม (ยืนยันใช้ร่วมกัน)

| ตาราง | ความหมายในระบบ | ใช้เมื่อไหร่ |
|--------|------------------|---------------|
| **`customer`** | ลูกค้าปลายทาง / ผู้ว่าจ้างซ่อมหรือซื้อบริการ | วางแผนขาย, การตลาด B2B, รายงานลูกค้า |
| **`customer_contact`** | ผู้ติดต่อหลายคนต่อหนึ่งรหัสลูกค้า (`cus_code`) | อีเมล/โทรติดตาม, แคมเปญ |
| **`job_data`** | งานซ่อม/บริการ ผูก `customer_id` | ความถี่งาน, ประเภทอุปกรณ์, มูลค่าเชิงพฤติกรรม |
| **`market`** | **ซัพพลายเออร์ / ร้านค้าที่สั่งอะไหล่** ไม่ใช่ “ตลาด” หรือกลุ่มลูกค้า | ใบสั่งซื้อ (`bill_order.market`), สต็อก (`store_data.s_market`) |

## ทำไมต้องแยกให้ชัด

- ฟิลด์ **`bill_order.market`** เก็บ `market_id` ที่ชี้ไปยัง **แหล่งซื้อของ** (เช่น ร้านอิเล็กทรอนิกส์, ตัวแทนจำหน่าย) ไม่ได้บอกว่าลูกค้าอยู่ “ตลาดไหน” ในความหมายการตลาด
- ถ้านำตาราง `market` ไปทำ “กลุ่มลูกค้า” หรือ segment รายงานจะ **ผิดทั้งหมด** — จะกลายเป็นการวิเคราะห์ซัพพลายเออร์แทนลูกค้า

## แนวทางในรายงานและ UI (ถ้าปรับได้ในอนาคต)

- ใช้คำว่า **“ซัพพลายเออร์” / “แหล่งจัดซื้อ”** แทนคำว่า “market” ในหน้าจอภาษาไทยที่อ้างถึงตารางนี้
- รายงานฐานลูกค้าให้อิง **`customer`** + **`job_data`** + **`bill_quota`** (หรือใบแจ้งหนี้ตามที่นิยามทางบัญชี)

## อ้างอิงสคีมา
