Node.js บน cPanel และโปรเจกต์นี้
================================

เป้าหมาย
--------
- cPanel หลายโฮสต์ (ณ การตั้งค่าปัจจุบัน) รองรับ Node ได้สูงสุดที่สาย 22.x
- โปรเจกต์กำหนดใน package.json:  "engines": { "node": ">=22" }
  หมายถึง: ใช้ Node 22 ขึ้นไป (รวม 23, 24 เมื่อโฮสต์ / เครื่องพัฒนารองรับ)

ไฟล์ .nvmrc
-----------
- มีค่า "22" — ให้ nvm / fnm / asdf ใช้ major 22 ตรงกับ cPanel
- อัปเดตเลขในไฟล์นี้เมื่อโฮสต์รองรับ LTS ใหม่และทีมตกลงย้ายเวอร์ชัน

บน cPanel (Setup Node.js App)
------------------------------
1. เลือก Node.js เวอร์ชัน 22.x (หรือสูงสุดที่แผงให้เลือกได้)
2. Application root / URL ชี้ไปที่โฟลเดอร์ที่มี package.json นี้ (หรือ build แล้วชี้ไปที่ dist + start script ตามที่ deploy)
3. หลัง pull โค้ดใหม่: รัน npm ci หรือ npm install ในโฟลเดอร์แอปตามขั้นตอนโฮสต์

CI / GitHub Actions
-------------------
- .github/workflows ใช้ node-version สอดคล้องกับโปรเจกต์ (ดู workflow ไฟล์)
- เมื่อย้าย major Node: แก้ .nvmrc, workflow, แล้วรัน build ทั้งเครื่องและ CI

หมายเหตุ
--------
- ไม่ผูกเวอร์ชันไว้ที่แค่ 22.x เดียว (เช่น 22.0.0) เพื่อให้อนาคตอัปเดต patch/minor ได้
- ถ้าต้องการล็อกเฉพาะสาย 22 จนกว่า cPanel จะรองรับสูงกว่า ให้ใช้เครื่องมือของโฮสต์เลือก 22.x และทีมยังคง ">=22" ใน package.json ได้
